# Payments & Orders

Backend-сервис учета платежей по заказам с учетом банковского статуса.

Проект моделирует бизнес-логику обработки платежей с разделением:
- учетного жизненного цикла платежа (в системе)
- банковского жизненного цикла (внешняя проверка статуса)

---

## 🧠 Архитектура

Проект построен с использованием принципов DDD:

- **Domain layer** — бизнес-логика (Order, Payment, Value Objects)
- **Application layer** — use cases
- **Infrastructure layer** — SQLAlchemy, репозитории, UoW

---

## ⚙️ Реализовано

- доменная модель (Order как aggregate root, Payment)
- value objects (Money, IDs)
- разделение банковского и учетного состояния платежа
- Unit of Work (SQLAlchemy реализация)
- репозитории с маппингом domain ↔ ORM
- интеграция с внешним банковским API (проверка статуса платежа)
- обработка ошибок доменного и инфраструктурного уровня
- unit и integration тесты (включая транзакции и rollback)
